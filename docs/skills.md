# Skills

## sdd-brainstorm command

- `sdd-propose` → creates the destination artifacts: PRD, OpenSpec proposal/design/specs, accepted assumptions, non-goals, risks, and open HITL questions.
- `sdd-spec` → creates the journey artifacts: vertical-slice Kanban/DAG tasks with blockers, unblocks, context packets, verification commands, and `[HITL]` or `[AFK]` labels.
- `sdd-design` → creates the design artifacts: architecture diagrams, data models, API contracts, and design constraints.
- `sdd-tasks` → creates the task artifacts: vertical-slice Kanban/DAG tasks with blockers, unblocks, context packets, verification commands, and `[HITL]` or `[AFK]` labels.

## sdd-ui-design

- `sdd-ui-design` → develops UI and UX direction, previews, design constraints, and design artifacts.

## sdd-??

- `sdd-test` → runs tests and captures evidence tied to success criteria.
- `sdd-security` → performs a deeper security pass when needed.
