## When to Use

Use this skill when Skillgrid needs a compact, project-local index of available skills and project convention files. The registry lets parent sessions pass resolved rules to subagents without forcing every subagent to rediscover the whole skill catalog.

## Canonical Output

Write the registry to:

```text
.skillgrid/project/SKILL_REGISTRY.md
```

This file is generated project context. It complements `docs/05-skills.md`; it does not replace the human-facing skills documentation.

When documenting or creating skills, use **`.agents/skills/_shared/SKILL-authoring-template.md`** as the canonical `SKILL.md` shape (triggers, out of scope, stop conditions, example prompt).

## What To Scan

Scan all existing skill roots that apply to the project:

- `.agents/skills/`
- `.cursor/skills/`
- `.claude/skills/`
- `.gemini/skills/`
- `.opencode/skills/`
- `.kilo/skills/`
- `skills/`
- user-level skill roots when accessible and relevant, such as `~/.cursor/skills/` or `~/.agents/skills/`

Also scan project convention files when present:

- `AGENTS.md` (project root, if any)
- `.configs/AGENTS.md`
- `.agents/rules/*.mdc`
- `CLAUDE.md`
- `GEMINI.md`
- `.cursorrules`
- `.github/copilot-instructions.md`
- `.cursor/rules/`

Skip generated mirrors only when they duplicate the Cursor source of truth exactly.

## Registry Format

Use this structure:

```markdown
# Skillgrid Skill Registry

Generated by `/skillgrid-init` or `/skillgrid-session`.

## Use

Parent sessions read this registry before delegating subagents. Subagents may receive the relevant compact rules directly in their prompt.

## Skills

| Skill | Trigger | Path |
|---|---|---|
| `<name>` | <short trigger> | `<path/to/SKILL.md>` |

## Compact Rules

### <skill-name>

- <5-15 concise, actionable rules>

## Project Conventions

| File | Role |
|---|---|
| `<path>` | <why it matters> |

## Refresh Notes

- Last refreshed: <ISO date>
- Refresh when skills, command files, personas, or project convention files change.
```

Compact rules should be extracted from the skill's critical patterns and hard constraints. Do not copy full skill bodies or long examples into the registry.

## Engram Persistence

If Engram is available, save only a concise pointer:

```text
title: Skillgrid skill registry
topic_key: skillgrid/skill-registry
type: config
content:
  What: Refreshed `.skillgrid/project/SKILL_REGISTRY.md`.
  Why: Subagents need stable compact rules without broad rediscovery.
  Where: `.skillgrid/project/SKILL_REGISTRY.md`
  Learned: Refresh when skill or convention files change.
```

Do not paste the full registry into Engram unless the project explicitly uses `artifactStore.mode: engram`.

## Subagent Use

Before launching a Skillgrid subagent:

1. Read `.skillgrid/project/SKILL_REGISTRY.md` if present.
2. Select only compact rules relevant to the delegated task.
3. Paste those rules into the subagent prompt under `## Project Standards`.
4. Include the registry path as an artifact reference, but do not ask the subagent to rediscover all skills.

## Commands

No fixed shell command is required. Use file reads, globbing, and exact search to build the registry from local artifacts.
